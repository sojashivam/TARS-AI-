API Documentation

Integrate PDFChat's capabilities into your own applications with our API. This documentation will guide you through using our Mistral AI-powered PDF and image analysis API.

Authentication

To use the PDFChat API, you'll need an API key. Sign up for an account and generate your API key from the dashboard.

Include your API key in the header of each request:

Authorization: Bearer YOUR_API_KEY

Endpoints

Upload Document

POST /api/v1/documents

Upload a PDF or image file for processing.

Request: Multipart form data with 'file' field

Response:

{
"document_id": "doc_12345",
"status": "processing",
"filename": "example.pdf"
}

Ask Question

POST /api/v1/documents/{document_id}/query

Ask a question about a previously uploaded document.

Request Body:

{
"query": "What is the main conclusion of the paper?"
}

Response:

{
"answer": "The main conclusion of the paper is that...",
"confidence": 0.92
}

Rate Limits

Our API has the following rate limits:

• Free tier: 100 requests per day
• Basic tier: 1,000 requests per day
• Professional tier: 10,000 requests per day

If you exceed these limits, you'll receive a 429 Too Many Requests response.

Error Codes

The API may return the following error codes:

• 400: Bad Request - Check your request parameters
• 401: Unauthorized - Invalid API key
• 404: Not Found - Document not found
• 429: Too Many Requests - Rate limit exceeded
• 500: Internal Server Error - Please contact support
